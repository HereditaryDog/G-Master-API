# 更新日志

本文档记录 `G-Master API` 对外版本的主要变化。

## v1.0.0-rc.10-GM.3 - 2026-06-10

### 变更

- 重新设计控制台令牌管理页面，新增令牌总数、启用数量、无限额度和已选令牌指标摘要，桌面端表格、批量操作和筛选区域更适合高频管理。
- 重新设计控制台使用日志页面，新增请求量、成功率、消耗和 Token 指标摘要，并统一查询条件、日志表格和详情展开的视觉层级。
- 令牌管理和使用日志在移动端切换为卡片化扫描体验，保留关键状态、消耗、时间、模型和操作入口，减少横向表格溢出。

## v1.0.0-rc.10-GM.2 - 2026-06-10

### 变更

- 操练场对话区新增顶部状态栏、模型/分组会话摘要、固定底部输入区和可隐藏调试面板，长对话与调试内容可以独立滚动。

### 文档

- 更新 README 当前稳定基线、版本号、Apifox 导出版本和本版本发布说明。

## v1.0.0-rc.10-GM.1 - 2026-06-09

### 新增

- 新增余额购买订阅接口 `POST /api/subscription/balance/pay`，用户可直接使用账户余额购买订阅套餐，订单会记录为 `balance` 支付来源。
- 新增 `NOTICE` 与 `THIRD-PARTY-LICENSES.md`，并在 Docker 镜像中保留许可证、NOTICE、致谢与第三方依赖清单。
- 使用日志移动端新增卡片视图，手机宽度下更容易扫描模型、费用、Token、耗时、Request ID 和展开详情。

### 变更

- 同步 v1 RC 上游 relay 稳定性更新：图片生成保留 `auto/high/medium/low` 等 quality 参数，大型 base64 请求体改用 body storage 包装并回填上游 `ContentLength`，降低内存峰值。
- Claude/Gemini 流式工具调用转换补齐并发 tool-call 索引、Gemini-to-Claude stop/final usage 兼容和多媒体内容拼接内存优化。
- 使用日志的模型名和用户名筛选改为“默认精确匹配，显式 `%` 才通配”，令牌名保持精确匹配。
- 渠道测试改为使用当前登录用户或 root 用户兜底，不再硬编码用户 `1`；复制渠道改为直接插入单个 clone，保证返回 ID 正确。
- 多 Key 渠道自动禁用逻辑会识别找不到当前 key、全部 key 禁用和 key 恢复场景，并在内存缓存状态变化时同步渠道状态。
- Waffo Pancake checkout 创建时写入本地 `trade_no` 到 `orderMerchantExternalId`，webhook 优先用该字段映射订单，同时保留旧 `orderId` 回退。
- 渠道批量模型测试不再为每个模型重复弹 toast，只保留批量进度和汇总；模型映射提交前校验必须是 JSON 对象且目标模型为字符串。

### 修复

- 上游异常响应体写入本地日志前会截断预览，避免超大错误 body 冲击日志存储；返回给调用方的机器可读错误结构保持完整。
- 修复多 Key 渠道找不到当前 key 时可能误伤第一个 key 的问题。
- 修复 Waffo Pancake webhook 使用平台内部 `ORD_*` 时无法稳定找到本地订阅/充值订单的问题。
- 修复异步图片任务进入通用任务轮询后可能被错误平台轮询器处理的问题，并保持图片生成上游路径为 `/v1/images/generations`。

### 文档

- 更新 rc.10 更新清单、README 当前稳定基线、Apifox 导出版本和 Docker 镜像授权文件保留策略。

## v1.0.0-rc.8-GM.2 - 2026-05-26

### 新增

- 为 Gaster Code 桌面端内置账号中心新增后端接口：套餐列表、checkout 创建与轮询、交易记录、订阅取消和订阅恢复。
- `/api/gaster-code/me` 扩展返回钱包余额、低余额状态、订阅条目、可用模型、可用功能和权益到期时间，便于桌面端直接展示账号中心。

- 交易记录接口会同时汇总充值、订阅订单、使用日志、退款和管理员额度调整记录。

### 文档

- 更新 Gaster Code Desktop Auth API 文档，补充账号中心接口清单、成功/失败 JSON 示例、稳定错误码和桌面端对接说明。
- 更新版本号、README 当前稳定基线和 Apifox 导出版本到 `v1.0.0-rc.8-GM.2`。

## v1.0.0-rc.8-GM.1 - 2026-05-25

### 新增

- 新增 Waffo Pancake 管理端目录能力，管理员可在支付设置页拉取店铺与产品目录，并一键创建、绑定 G-Master API 使用的店铺和一次性充值产品。
- 新增 Waffo Pancake 配置原子保存接口，商户、私钥、回跳地址、店铺和产品绑定会作为一组配置写入，避免只保存部分字段导致支付入口不可用。
- 通道模型测试失败时展示后端返回的错误码，`model_price_error` 等可定位错误会在弹窗中直接显示，并继续提供跳转到倍率设置页的入口。

### 变更

- 请求分发路径改为快速读取 JSON 请求体中的 `model` 和 `group` 元数据，读取后会重置 body，确保后续 relay 仍能完整读取原始请求。
- relay 的禁用字段移除逻辑增加快速判断，请求体没有受控字段时不再进入完整 JSON 解析流程。
- OpenAI 兼容流式响应的 token 解析改为边读边汇总，减少长流式响应的内存占用，并继续通过项目统一 JSON 封装解析。
- 模型价格编辑器会规避浮点显示漂移，价格、补全、缓存、图片和音频相关数值展示更稳定。
- 内部支付引用和兜底邮箱统一改为 G-Master API 命名。

### 修复

- 阻止通过通用配置接口直接写入支付合规确认字段，邀请奖励额度在未确认支付合规前也不能被设置为正数。
- 保留模型健康度对空分组使用日志的兜底聚合，避免已有真实调用后控制台仍显示“暂无性能数据”。
- 保留使用日志的转义 contains 筛选行为，避免 `%`、`_` 等字符造成误匹配。

### 文档

- 补充 rc.8 更新清单与实施计划，记录本次 G-Master API 的选择性迁移范围、保留项和验证矩阵。
- 新增本版本发布说明，便于 GitHub Release 直接引用。

## v1.0.0-rc.7-GM.2 - 2026-05-22

### 修复

- 修复异步图片生成任务在上游网关返回 `504`、Cloudflare `524` 或 `upstream_timeout` 时不会重试的问题。队列 worker 现在至少保留一次超时重试机会，避免复杂提示词长耗时生成被单次网关超时直接判失败。
- 修复异步图片生成后台任务复用原请求上下文时可能携带已关闭或陈旧 request body storage 的问题。每次渠道尝试都会从保存的请求字节重建请求体，确保提示词、尺寸和模型参数原样送到上游。
- 修复异步图片任务重试前响应 recorder 未清空时可能保留上一轮错误响应的问题。
- 修复异步图片任务创建阶段未初始化 `ChannelMeta` 时访问 `relayInfo.ChannelId` 导致的 nil pointer panic；任务初始化现在只在 `ChannelMeta` 存在时读取渠道 ID，并在入口 panic 时返回稳定的 OpenAI-style 错误和请求 ID。

### 文档

- 补充图片生成异步任务文档，明确客户端应保留原始提示词、创建任务后轮询状态，并在上游超时失败时读取机器可读错误结构。

## v1.0.0-rc.7-GM.1 - 2026-05-22

### 新增

- 新增顶部导航/状态接口的模块开关控制，用户端模块展示会跟随后台配置，同时保留 `Gaster Code` 入口。

- 模型列表会优先展示启用渠道的供应商归属信息，避免 `owned_by` 与实际可用渠道脱节。
- 性能指标摘要只纳入当前有效分组，同时继续保留使用日志兜底，避免模型健康度在真实调用后仍为空。
- 参数覆写审计扩大到敏感来源路径，覆盖 `input`、`messages`、`system`、`instructions` 等上游请求体字段。
- 运营设置页的渠道亲和编辑器补充请求头来源选项、占位示例和 JSON 模板示例。
- 使用日志的模型、用户名和令牌名筛选改为转义后的 contains 匹配，避免 `%`、`_` 等字符造成误匹配。
- Debug 日志改为按需格式化，生产环境关闭 Debug 时不再提前拼接大请求体日志。

### 修复

- 修复 Waffo Pancake 充值相关路由未启用的问题，并将 webhook 可用性从普通充值商品配置中拆分出来，便于订阅和充值共用基础配置。
- 订阅订单写入充值历史时补充 `payment_provider`，避免后续支付渠道审计信息缺失。
- 修复渠道列表在未输入搜索关键词时忽略分组筛选的问题，类型统计也跟随分组过滤。
- 修复管理员删除用户失败时仍返回成功的问题。
- `/api/status` 补充注册开关，登录页会在关闭注册时隐藏注册入口。
- 管理员新增用户时前端补充 8 到 20 位密码长度校验。

## v1.0.0-rc.5-GM.2 - 2026-05-15

### 修复
