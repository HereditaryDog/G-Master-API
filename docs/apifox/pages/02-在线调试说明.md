# 在线调试说明

你可以使用文档调试台、本地 API 客户端或任意 HTTP 工具直接发起请求测试。

## 建议环境变量

- `base_url=https://gmapi.fun`
- `openai_base_url=https://gmapi.fun/v1`
- `api_key=你的 sk-令牌`
- `access_token=用户访问令牌`
- `new_api_user=当前登录用户 ID`

## 调试模型接口

1. 打开任意 `/v1/*` 接口，例如 `POST /v1/chat/completions`
2. 选择环境
3. 设置请求头：
   - `Authorization: Bearer {{api_key}}`
4. 填写请求体
5. 点击发送

## 调试系统接口

调试 `/api/*` 用户接口时，通常需要：

- `Authorization: Bearer {{access_token}}`
- `New-Api-User: {{new_api_user}}`

如果使用 `Session` 登录态调试，也可以改用浏览器 Cookie。

## 典型问题

- `Invalid authorization header`
  通常是 `Bearer` 前缀缺失或令牌无效。
- `User id mismatch`
  说明 `New-Api-User` 与当前访问令牌所属用户不一致。
- `404 / 无法访问`
  优先检查环境中的 `base_url` 是否配置正确。
- 请求到了默认官方地址
  说明客户端仍在使用默认 Base URL，需要切换到 `https://gmapi.fun/v1` 或 `https://gmapi.fun/api`。
