# 发出请求

## 模型接口的标准写法

所有 OpenAI 兼容模型接口都建议使用：

- Base URL：`https://gmapi.fun/v1`
- Header：`Authorization: Bearer sk-your-token`

## 最常见的请求头

### 模型中转接口

```http
Authorization: Bearer sk-your-token
Content-Type: application/json
```

### 用户中心接口

```http
Authorization: Bearer your-access-token
New-Api-User: your-user-id
Content-Type: application/json
```

## Curl 示例

```bash
curl https://gmapi.fun/v1/chat/completions \
  -H "Authorization: Bearer sk-your-token" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-chat-model",
    "messages": [
      {
        "role": "user",
        "content": "你好"
      }
    ]
  }'
```

其中 `your-chat-model` 请替换为控制台可见、且当前令牌允许访问的模型名。

## 常见调用路径

- `GET /v1/models`
- `POST /v1/chat/completions`
- `POST /v1/responses`
- `POST /v1/embeddings`
- `POST /v1/images/generations`
- `POST /v1/audio/transcriptions`
- `POST /v1/video/generations`

## 常见错误排查

- `401 Unauthorized`
  令牌缺失、无效，或 `Bearer` 前缀未带上。
- `429 Too Many Requests`
  额度不足、模型受限或频率过高。
- `404 Not Found`
  基址配置错误，或者请求了当前服务未开放的路径。
