# 帮助中心

## FAQ

### 1. `Invalid authorization header`

通常是这几种情况：

- 没有加 `Bearer ` 前缀
- 填错了 `sk-` 令牌
- 调用了用户中心接口，却使用了模型令牌

### 2. 调用 `/api/*` 接口时报用户不匹配

需要补齐：

- `Authorization: Bearer 访问令牌`
- `New-Api-User: 当前用户 ID`

### 3. 为什么文档调试台或浏览器调试时提示跨域、插件或权限限制

优先检查：

- 是否使用了正确环境变量
- 是否缺少浏览器调试所需插件或授权
- 是否把请求直接发到了错误的默认地址

### 4. 常见 HTTP 状态码

- `200`：请求成功
- `400`：参数错误
- `401`：鉴权失败
- `403`：权限不足
- `404`：路径不存在
- `429`：频率或额度限制
- `500`：服务内部错误或上游异常

### 5. 如何确认当前网关版本

请求：

```bash
curl https://gmapi.fun/api/status
```

返回中的 `data.version` 即当前版本。
