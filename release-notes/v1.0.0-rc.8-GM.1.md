# v1.0.0-rc.8-GM.1

## Highlight

本版本继续完善 G-Master API 的 v1 RC 控制台与支付能力，重点补齐 Waffo Pancake 店铺/产品绑定、支付合规加固、请求路径性能优化、模型价格显示归一化和通道测试错误码展示。

## 主要更新

- Waffo Pancake 支付设置页新增目录拉取、店铺选择、产品选择和一键创建绑定能力。
- Waffo Pancake 商户、私钥、回跳地址、店铺和产品绑定改为原子保存，减少配置半更新风险。
- 通道模型测试失败时展示机器可读错误码，便于直接定位模型价格、渠道权限或上游异常。
- 请求分发和 relay 禁用字段处理增加快速路径，降低常规 OpenAI 兼容请求的解析开销。
- OpenAI 兼容流式 token 汇总改为边读边解析，减少长流式响应的内存压力。
- 模型价格编辑器规避浮点漂移，价格、补全、缓存、图片和音频数值展示更稳定。

## 问题修复

- 阻止通用设置接口绕过支付合规确认字段。
- 未确认支付合规前，邀请奖励额度不能被设置为正数。
- 保留模型健康度的使用日志兜底聚合，避免真实调用后仍无数据。
- 保留使用日志转义 contains 筛选，避免特殊字符造成误匹配。
- 内部支付引用和兜底邮箱统一使用 G-Master API 命名。

## 安装说明

- 新部署可直接按 `README.md` 的 Docker Compose 或源码构建流程安装。
- 已部署实例升级前建议备份数据库、`data/` 和 `logs/` 目录。
- 升级后建议进入控制台检查支付设置、模型价格、通道测试、使用日志和 Gaster Code 页面。
