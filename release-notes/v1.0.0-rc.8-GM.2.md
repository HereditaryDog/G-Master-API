# v1.0.0-rc.8-GM.2

## Highlight

本版本补齐 Gaster Code 桌面端内置账号中心所需的 G-Master API 后端接口，让桌面端可以用同一套 PKCE 登录态展示账号、钱包、订阅、交易记录，并发起充值或订阅管理。

## 主要更新

- `/api/gaster-code/me` 扩展返回钱包、订阅、权益、可用模型和可用功能。
- 新增 `/api/gaster-code/billing/plans`，返回可用于桌面端展示的充值与订阅套餐。
- 新增 `/api/gaster-code/billing/checkout` 和 `/api/gaster-code/billing/checkout/:id`，桌面端只打开网页 checkout URL，支付密钥仍留在 G-Master API 网页流程中。
- 新增 `/api/gaster-code/billing/transactions`，统一返回充值、订阅、使用、退款和额度调整记录。
- 新增 `/api/gaster-code/subscription/cancel` 与 `/api/gaster-code/subscription/resume`，支持取消到期停用和可恢复订阅恢复。
- 新增 checkout 持久化表，支持桌面端轮询状态，并能根据用户后续网页充值或订阅成功记录推断为 `paid`。

## 问题修复

- Gaster Code 桌面端认证失败统一返回 `GMASTER_AUTH_EXPIRED`，同时保留 `legacy_code=authentication_failed` 兼容旧客户端。
- 修复轻量测试环境中保留字分组列名未初始化时可能生成空 `SELECT` 的问题。
- 确认 provider token 在用户充值或订阅导致分组变更后仍会同步到最新用户分组。

## 安装说明

- 新部署可直接按 `README.md` 的 Docker Compose 或源码构建流程安装。
- 已部署实例升级前建议备份数据库、`data/` 和 `logs/` 目录。
- 升级后建议检查 `/api/status` 版本、Gaster Code 登录态、`/api/gaster-code/me`、billing plans、checkout 轮询、交易记录和 provider token 返回。
